Proof of concept for the server API and data model. This can actually be played over the server. For a one-person test you have to use two different browsers and register two different users.

The idea is to avoid any complexity and to start with a simple number guessing game. But the implementation has all the elements of the planned "big" game:

"Prepare" - Player enters a number from 1-100. He submits the number to the server, where to same processing takes place as in the actual game (including validation), except the payload is not an array of ships but a plain number.

"Match" - A list of ready players is displayed and two players can form a pair of oppenents. Almost the same as in the actual game from a data perspective.

"Battle" - Guessing instead of shooting

As a proof of concept this is reduced to the minimum: no tests, no services, only rudimentary ui, error handling and validation.
